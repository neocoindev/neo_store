# Решение проблемы доступа к GitHub репозиторию

## Текущая проблема
- Токен аутентифицирует, но нет прав на запись в `neocoindev/neo_store.git`
- Ошибка: `Permission denied`

## Возможные решения:

### Решение 1: Проверить права токена

Убедитесь, что токен имеет права `repo`:
1. GitHub → Settings → Developer settings → Personal access tokens
2. Найдите ваш токен
3. Проверьте, что отмечен scope `repo` (все права репозиториев)
4. Если нет - создайте новый токен с правами `repo`

### Решение 2: Проверить доступ к репозиторию

Если репозиторий `neocoindev/neo_store` принадлежит организации:
1. Убедитесь, что вы являетесь членом организации `neocoindev`
2. Проверьте, что у вас есть права на запись в репозиторий
3. Если репозиторий приватный, убедитесь, что токен имеет доступ к организации

### Решение 3: Создать репозиторий (если его нет)

Если репозиторий не существует:
1. Перейдите на https://github.com/new
2. Владелец: выберите `neocoindev` (если это организация)
3. Название: `neo_store`
4. Создайте репозиторий
5. Затем выполните push

### Решение 4: Использовать GitHub CLI

Установите GitHub CLI и авторизуйтесь:
```bash
# Установите GitHub CLI (если еще не установлен)
# Затем авторизуйтесь
gh auth login

# Создайте репозиторий (если нужно)
gh repo create neocoindev/neo_store --public

# Запушите код
git push -u origin main
```

### Решение 5: Проверить настройки организации

Если `neocoindev` - это организация:
1. Перейдите в настройки организации
2. Settings → Third-party access → Personal access tokens
3. Убедитесь, что токены разрешены для организации
4. Может потребоваться одобрение администратора организации

## Быстрая проверка

Выполните команду для проверки доступа:
```bash
curl -H "Authorization: token YOUR_TOKEN" https://api.github.com/repos/neocoindev/neo_store
```

Если получите 404 - репозиторий не существует
Если получите 403 - нет прав доступа
Если получите 200 - репозиторий существует и доступен

## Альтернатива: Создать репозиторий под вашим аккаунтом

Если проблема не решается, создайте репозиторий под вашим аккаунтом `tazhibaevnurs`:

```bash
# Удалите старый remote
git remote remove origin

# Добавьте ваш репозиторий
git remote add origin https://github.com/tazhibaevnurs/neo_store.git

# Создайте репозиторий на GitHub, затем:
git push -u origin main
```
